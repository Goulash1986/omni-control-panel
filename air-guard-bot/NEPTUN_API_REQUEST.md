## Тема:

## Текст листа:

Доброго дня, команда NEPTUN!

Передусім хочу подякувати за роботу, яку ви робите. Ваш сервіс реально рятує життя — і саме тому я звертаюся.

Я розробляю **приватний некомерційний бот** для однієї конкретної людини — моєї дівчини, яка живе в Дніпрі (Новокодацький район, ж/м Червоний Камінь). Мета бота — агрегувати інформацію про повітряні загрози з кількох джерел (alerts.in.ua, OSINT-канали в Telegram) і доставляти їй персоналізовані сповіщення з урахуванням її точної геолокації, з real-time картами загроз та автоматичним голосовим дзвінком у разі критичної ситуації.

Зараз архітектура працює на парсингу OSINT-каналів та офіційному API alerts.in.ua, але **позиційні дані у вашому сервісі набагато точніші та оперативніші**. Я бачу, що ви оновлюєте карту кожні 5 секунд із real-time трекінгом шахедів, крилатих та балістики — це саме те, чого нам бракує для побудови якісної системи раннього попередження.

**Що саме мені потрібно:**
- Доступ до stream'а актуальних позицій загроз у Дніпропетровській області (область / громада Дніпро)
- Бажано через WebSocket, SSE, або REST polling (~5 сек)
- Дані: тип загрози, координати, напрямок руху, час останнього оновлення

**Обмеження, з якими я готовий працювати:**
- Rate limit (наприклад 1 запит / 5 сек) — повністю влаштовує
- Тільки моя область (Дніпропетровська) — повністю влаштовує
- Trial доступ з обмеженням часу — приймається
- Будь-які умови щодо невикористання даних поза призначенням — підпишу і дотримаюсь

**Що я готовий запропонувати взамін:**
- Атрибуція **«Powered by NEPTUN»** на кожній згенерованій карті в боті
- Зворотний зв'язок: репорти про точність даних, помилки, edge cases, які можемо знайти в реальному використанні
- Технічна співпраця: якщо вам цікаво — можу поділитися моєю системою geo-нормалізації (резолв синонімів районів та мікрорайонів Дніпра), яку я зробив для бота
- Готовність бути бета-тестувальником будь-яких нових фіч
- Невелика донація на підтримку проєкту, якщо це доречно (можу пожертвувати на ПС ЗСУ за вашою рекомендацією)

**Чого це НЕ є:**
- Це **не комерційний продукт**, я не планую монетизацію
- Це **не публічний бот** — він обмежений двома Telegram ID (моя дівчина + я як backup-канал)
- Я **не планую реселити** ваші дані або робити похідний публічний продукт

Якщо у вас є **публічна стороння політика API** з якоюсь формою — я з радістю заповню. Якщо потрібна **верифікація** хто я і навіщо мені це — готовий надати документи, підтвердити проживання дівчини в Дніпрі, надати посилання на свої проєкти, тощо.

## Поради до відправки (оновлено May 2026)

1. **Канал зв'язку:** через Telegram-канал NEPTUN (на сайті neptun.in.ua вказано тільки tg, email відсутній). Шукати @username каналу у шапці сайту, тоді — direct message адміну або через коментарі / групу обговорення якщо вони відкриті
2. **Підпис** з реальним ім'ям. Без анонімності
3. **GitHub-посилання** на свій код — обов'язково. Підтверджує що ти розробник, а не агрегатор-конкурент
4. **Не очікуй швидкої відповіді** — це маленька команда волонтерів. Тиждень-два — норма
5. **Якщо тиша 2 тижні** — можна тактовно нагадати одним коротким повідомленням
6. **Альтернативна стратегія якщо мовчать:** працювати з їх Telegram-каналом як OSINT-джерелом (Tier-2 confidence ~0.85) — це працює без жодних API

## Що робити поки чекаєш

Не блокуй розробку на NEPTUN — у нас є alerts.in.ua API (працює), OSINT-канали (працюють), Dron Alerts SSE (теж публічне). NEPTUN додамо як ще одне джерело, коли (і якщо) дадуть доступ. Архітектура з самого початку розрахована на додавання нових джерел через `sources.yaml` без переписування коду.
